## вҡЎ What is Reach?

Most job seekers spend **hours** doing this manually:

```
Search LinkedIn вҶ’ Find recruiter вҶ’ Copy email вҶ’ Open Gmail вҶ’ Write email вҶ’ Attach resume вҶ’ Send вҶ’ Repeat
```

**Reach collapses that entire pipeline into a single form submission.**

```
Fill form вҶ’ Click Start вҶ’ Watch it happen вҶ’ Done.
```

You enter your Gmail, an App Password, a job keyword, and upload your resume.  
Reach handles everything else вҖ” browser automation, scraping, composing, and sending.

## рҹҡҖ Getting Started

### Prerequisites

- Node.js `v18+`
- Gmail account with **2-Step Verification** enabled
- A **Google App Password** ([How to get one вҶ“](#-google-app-password-setup))

### Installation

```bash
# 1. Clone the repo
git clone https://github.com/YOUR_USERNAME/reach.git
cd reach

# 2. Install backend dependencies
cd server
npm install

# 3. Install frontend dependencies
cd ../client
npm install
```

### Run

```bash
# Terminal 1 вҖ” Backend
cd server
node server.js
# вң“ Server running at http://localhost:3000

# Terminal 2 вҖ” Frontend
cd client
npm run dev
# вң“ Open http://localhost:5173
```

## рҹ§  Engineering Challenges

Building this wasn't straightforward. Here are the real problems solved:

<details>
<summary><b>рҹӨ– LinkedIn Bot Detection</b></summary>

<br/>

LinkedIn checks `navigator.webdriver` in JavaScript to detect Puppeteer.

```js
// Run before any page loads вҖ” hides the Puppeteer fingerprint
await page.evaluateOnNewDocument(() => {
  Object.defineProperty(navigator, "webdriver", { get: () => undefined });
});

// Also spoof the User-Agent
await page.setUserAgent("Mozilla/5.0 (Windows NT 10.0; Win64; x64)...");
```

</details>

<details>
<summary><b>рҹҺҜ Selector Stability вҖ” LinkedIn uses random class names</b></summary>

<br/>

LinkedIn generates random IDs and class names on every page load:

```html
<!-- These change every session вҖ” useless as selectors -->
<input id=":r3:" class="efb8f1b0 _122158cd _7d9cda5f ...">
```

The fix: target **semantic HTML attributes** that LinkedIn cannot randomize:

```js
// Stable вҖ” tied to HTML/accessibility spec
"input[autocomplete='username webauthn']"   // email
"input[autocomplete='current-password']"    // password
```

</details>

<details>
<summary><b>вҢЁпёҸ The Tab Key Solution (hardest bug)</b></summary>

<br/>

The password field was consistently not found by Puppeteer вҖ” even though it was visible in the DOM. LinkedIn's React re-renders the form after email input loses focus, and the password field's event listeners aren't attached yet.

Tried and failed:
- `waitForSelector` with multiple selectors
- `scrollIntoView` + `focus` + 2000ms delay  
- Targeting `autocomplete='current-password'` directly

**The fix that worked:**

```js
await emailField.type(LINKEDIN_EMAIL, { delay: 80 });

// Press Tab вҖ” exactly like a real user would
// LinkedIn's own JS moves focus to the password field
await page.keyboard.press("Tab");
await new Promise((r) => setTimeout(r, 1000));

// Type directly into the now-focused field вҖ” no selector needed
await page.keyboard.type(LINKEDIN_PASSWORD, { delay: 80 });
```

> Mimicking real user behavior was the solution. No selector. Just `Tab`.

</details>

<details>
<summary><b>рҹ”„ Browser Lifecycle вҖ” Chromium staying open after crashes</b></summary>

<br/>

Early version had `browser.close()` after a `return` statement вҖ” unreachable code. Puppeteer processes were piling up.

```js
// вқҢ browser.close() never runs
return { success: true };
await browser.close(); // dead code

// вң… finally block ALWAYS runs вҖ” success or failure
try {
  // automation...
  return { success: true };
} catch (err) {
  return { success: false };
} finally {
  await browser.close(); // guaranteed cleanup
}
```

</details>

<details>
<summary><b>вҡӣпёҸ React Crash вҖ” "Objects are not valid as a React child"</b></summary>

<br/>

Axios automatically parses JSON responses. `res.data` was already the object `{ success, message }` вҖ” not a string. Setting that as `status.message` made React try to render an object as a DOM text node.

```js
// вқҢ res.data is { success: true, message: "..." } вҖ” React can't render objects
setStatus({ type: "success", message: res.data });

// вң… Extract just the string
setStatus({ type: "success", message: res.data.message });
```

</details>

## рҹ”’ Security

- вң… **Zero credential storage** вҖ” Gmail and App Password exist only in memory during the request
- вң… **App Password scoped** вҖ” not your main Gmail password, revocable anytime
- вң… **No database** вҖ” nothing is persisted beyond the uploaded resume file
- вң… **In-memory processing** вҖ” all sensitive data is garbage collected after the async function resolves
